## 核心玩法說明

### 遊戲類型
捕魚機（Fish Shooting Game）。玩家消耗子彈射擊螢幕上的目標物，擊中後獲得對應倍率的獎勵。

### 基本流程
```
玩家選擇角色（吉伊卡哇/小八/烏薩奇）
    ↓
設定下注金額（1-100 單位）
    ↓
點擊/觸碰螢幕射擊目標物
    ↓
Server 判定是否命中（基於 RTP 計算）
    ↓
命中 → 獲得獎勵（倍率 × 下注金額）
未命中 → 消耗子彈
    ↓
特殊事件觸發（Bonus/BOSS）
    ↓
繼續遊玩
```

## 遊戲狀態機

```
IDLE（等待）
    ↓ 玩家開始遊戲
MAIN_GAME（主遊戲）
    ├── 正常遊玩
    ├── BOSS_WARNING（BOSS 警告，3 秒）
    │       ↓
    │   BOSS_BATTLE（BOSS 戰）
    │       ↓ BOSS 擊敗或逃跑
    │   回到 MAIN_GAME
    └── BONUS_TRIGGER（Bonus 觸發）
            ↓
        BONUS_GAME（Bonus 遊戲）
            ↓ Bonus 結束
        回到 MAIN_GAME
```

## Bonus 遊戲設計

### 觸發條件
- 主遊戲中隨機觸發（每 50-100 局）
- 特定目標物組合觸發（待定義）

### Bonus 遊戲流程
1. 播放 `bonus_ready.wav`
2. 場景切換到 `BonusGame.tscn`
3. 播放 `bonus_game.wav`（BGM）
4. 玩家獲得固定次數的免費射擊（10-20 次）
5. 免費射擊期間 RTP 提升到 105-115%
6. Bonus 結束，顯示總獎勵
7. 返回主遊戲

### Bonus 遊戲數值
| 參數 | 值 |
|------|-----|
| 免費射擊次數 | 10-20 次（隨機）|
| Bonus RTP | 105-115% |
| 最大 Bonus 獎勵 | 1000x 下注金額 |

## BOSS 戰設計

### BOSS 出現條件
- 計時器：每 200-500 局隨機觸發
- 特殊條件：玩家連續命中 10 次（待實作）

### BOSS 戰流程
1. Server 發送 `boss_spawn` 訊息
2. Client 播放 `boss_warning.wav`（3 秒警告）
3. BOSS 出現，播放 `boss_enter.wav`
4. BOSS 有 HP 值（1000 點）
5. 玩家射擊 BOSS，每次命中扣除 HP
6. BOSS HP 歸零 → 擊敗，獲得大獎
7. BOSS 計時器到期（30 秒）→ BOSS 逃跑，無獎勵

### BOSS 數值（B001）
| 參數 | 值 |
|------|-----|
| HP | 1000 |
| 持續時間 | 30 秒 |
| 每次命中獎勵 | 10-50x |
| 擊敗獎勵 | 100-500x |
| 出現頻率 | 每 200-500 局 |

## 射擊判定邏輯

### Server 端判定
```
玩家發送 shoot 訊息
    ↓
Server 確認目標物存在
    ↓
計算命中機率（基於 RTP 目標值）
    ↓
隨機判定：命中 or 未命中
    ↓
回傳 hit or miss 訊息
```

### 命中機率計算
- 基礎命中率由 RTP 目標值反推
- 高倍率目標物命中率較低
- 低倍率目標物命中率較高
- 動態調整確保長期 RTP 符合目標
