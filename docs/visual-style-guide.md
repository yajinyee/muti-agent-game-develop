# 吉伊卡哇：像素大討伐 — 視覺風格指南

> 維護者：Art Director Agent  
> 版本：v1.0（2026-05-18）  
> 用途：確保所有美術資產的視覺一致性

---

## 1. 整體風格定義

### 風格關鍵字
- **16-bit Retro Pixel Art**（規格書 1章）
- **Chibi 可愛風格**（頭大身小，圓形輪廓）
- **IP 包裝**：吉伊卡哇官方色彩 + 像素化詮釋
- **明亮活潑**：高飽和度，清晰輪廓，強烈對比

### 禁止風格
- 寫實風格（不符合 IP 精神）
- 暗色系（遊戲主色調是明亮的）
- 模糊邊緣（必須是清晰的像素輪廓）
- 超過 16 色的調色板（每個角色/目標物）

## 9. 視覺一致性評分標準

| 分數 | 標準 |
|------|------|
| 90-100 | 所有資產符合風格指南，無明顯不一致 |
| 80-89 | 少數資產有輕微風格偏差，不影響整體 |
| 70-79 | 部分資產風格不一致，需要修正 |
| < 70 | 嚴重風格問題，禁止發布 |

### 當前評分：93/100
**扣分原因：**
- usagi 系列像素密度（58%）略低於 chiikawa/hachiware（64-66%）— -3分
- T105 金幣魚形狀細長（42%），雖然符合形狀特性，但視覺存在感略弱 — -2分
- 缺乏正式的美術審核報告（本文件建立前）— -2分

**提升方向：**
1. usagi 角色可考慮加粗輪廓或增加細節
2. T105 可考慮加大魚身比例，增加金色鱗片細節
3. 定期執行美術審核報告

## 10. 資產審核流程

```
Sprite Generation Agent 生成圖像
    ↓
Art Director 執行 process_sprites.py --mode qc
    ↓
確認：height diff ≤ 2px, width diff ≤ 4px
    ↓
確認：像素密度符合門檻
    ↓
確認：顏色符合官方調色板
    ↓
通過 → 寫入 reports/art/art-review-[DATE].md
    ↓
QA Agent 執行 qa_check.py 確認 Visual Consistency ≥ 90
```
