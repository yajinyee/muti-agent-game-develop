## 今日重大發現（2026-05-24）

### 根本問題：HUD.gd 有 Parse Error，遊戲從來沒有正常運作過

**發現過程：**
1. 用 Godot headless --import 測試，發現 HUD.gd 第 632 行有未閉合字串
2. 逐一修復後發現有 82 個 UI Panel 腳本損壞
3. 這些腳本從 git 歷史的第一個 commit 就已經損壞（縮排遺失、字串截斷）

**根本原因：**
- 中文字串在某些環境下被截斷，導致字串字面量未閉合
- 例如：`name_lbl.text = "??酋摮?` 沒有結尾引號
- 這導致整個 HUD.gd 無法解析，所有按鈕完全失效

**修復方式：**
- 重建了精簡版 HUD.gd（只保留核心功能）
- 移除了所有損壞的 Panel preload
- 重新 build HTML5（20:09 完成）

**教訓：**
- 每次修改 GDScript 後必須用 `godot --headless --import` 驗證
- 中文字串在 Windows 環境下容易損壞，應該用英文或確保 UTF-8 編碼
- 不能只靠 `go build` 驗證，Client 端也需要驗證
